Fetch

Fetch 和 Ajax 的区别

  1. 语法简洁，更加语义化
  2. 基于标准 Promise 实现，支持 async/await
  3. fetch()返回的 promise 将不会拒绝 http 的错误状态，即使响应是一个 HTTP404 或者 500
  4. 在默认情况下 fetch 不会接收或者发送 cookies

Fetch 解决不兼容 ie 的问题

fetch 的兼容性很差，大部分浏览器都不支持，幸运的是可以使用“whatwg-fetch”插件完美兼容 IE8+

安装 whatwg-fetch：

  • npm install whatwg-fetch -S
  • //导入(main.js)whatwg-fetch 兼容 IE8+浏览器
  • import “whatwg-fetch”

Axios

使用 axios 时挂载到全局（main.js）也就是 Vue 原型上面去

post 请求注意

axios 默认支持 raw(json)模式，对于 form 类型的接口，需要将 axios 发送的数据转换成 form 类型，有两种方法

  1. 使用 URLSearchParams

    但是 URLSearchParams 存在兼容性问题，解决办法：

    安装 url-search-params-polyfill

    1. npm i url-search-params-polyfill --save

    2. 在 src/main.js 文件中导入

      improt "url-search-params-polyfill" 兼容 IE

  2. 使用 qs

    1. 安装 qs

      npm i qs -S

    2. 导入 qs

Axios 请求配置

开发中 URL 的开发模式与生产模式会变化，所以将 URL 中要变化的部分取出封装到config.js中，再挂载到全局上

Axios 创建实例

使用axios.create()方法来创建一个 axios 实例，之后就使用该实例向服务端发起请求，就不用每次请求时重复设置配置选项了，相当于全局配置

之后再使用 axios 进行请求时就不需要再写baseURL里面的这一部分了
